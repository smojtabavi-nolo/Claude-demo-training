# Baseline Operating Rules

Use this template to define the default operating posture for the rest of the course.

## Investigation First

- We inspect before we edit.
- We gather evidence before we diagnose.

## Change Control

- We prefer bounded phases over large one-shot changes.
- We stop when the scope expands beyond the approved work package.

## Verification

- We run commands before claiming success.
- We capture test or CLI output as evidence.

## Approvals

- We ask for approval before destructive or high-risk actions.
- We do not assume permission to touch unrelated files or systems.

## Communication

- We pass artifacts, not long transcripts.
- We state what is confirmed, what is inferred, and what remains unknown.
